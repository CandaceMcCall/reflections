What happens when you initialize a repository? Why do you need to do it?
Need to initialize structure and metadata of repository

How is the staging area different from the working directory and the repository?
What value do you think it offers?
Staging area contains the files to be committed.   Enables you to have a place for all files to be placed so all can be committed in a logical fashion.

How can you use the staging area to make sure you have one commit per logical
change?
Only place the files you want to commit in the staging area.  Then commit all at one time.

What are some situations when branches would be helpful in keeping your history
organized? How would branches help?
If you are adding a experiemental feature and don't want to change the master codebase.

How do the diagrams help you visualize the branch structure?
Helps you to see branches and and commits that will be lost.

What is the result of merging two branches together? Why do we represent it in
the diagram the way we do?
Will add branch to master.   Diagram makes sure we see how structure changes and how commits flow.

What are the pros and cons of automatic merging vs. always doing merges
manually?
Automatic merging is good when adding new feature to part of master that was not being changed.   Manually merge more effort.
